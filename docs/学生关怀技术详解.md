# 学生关怀系统 - 技术详解

> 面向校园场景的学生关怀智能分析模块，通过多智能体协同分析六维度风险线索，辅助班主任更早发现需要关注的学生。

---

## 一、系统定位

| 是 | 不是 |
|----|------|
| ✅ 学生风险线索识别系统 | ❌ 心理诊断系统 |
| ✅ 班主任关怀辅助决策系统 | ❌ 医疗判断系统 |
| ✅ 基于多源数据的六维研判系统 | ❌ 自动给学生贴标签的系统 |

**核心输出**：六维风险雷达图 + 风险分数明细 + 综合研判结果

---

## 二、技术架构

```
事实采集层 → 信号抽取层 → 线性评分层 → 双层贝叶斯推断 → 多智能体解释 → 人在回路确认
     │            │            │              │                │              │
     ├─标签      ├─时间衰减   ├─加权求和    ├─辅助层(Odds-LR)  ├─6专家并行   ├─教师确认
     ├─成绩      ├─文本极性   └─归一化      └─网络层(因果图)  └─整合汇聚   └─证据更新
     ├─出勤      └─保护性证据
     ├─行为
     ├─观察
     ├─AI摘要
     └─图谱
```

## 三、核心技术原理

### 3.1 多智能体协同

**原理**：LangGraph编排6个专家智能体并行研判

**优势**：
- 并行执行，总耗时≈单个专家耗时
- 节点级容错，单个专家失败不影响整体
- 兜底机制，模型超时时使用规则引擎结果

### 3.2 双层贝叶斯推断

#### 第一层：辅助层（Odds-LR累乘）

用于四个高敏感维度（emotion/safety/family/social）：

```
后验几率 = 先验几率 × LR₁ × LR₂ × ... × LRₙ

混合公式：
FinalScore = λ × LinearScore + (1-λ) × Posterior
```

**边际递减机制**：处理证据相关性
```
第n个证据权重 = LR^(1/√n)
```

**动态先验机制**：根据线性分数调整先验
```
DynamicPrior = BasePrior + (1-BasePrior) × LinearScore × 0.3
```

#### 第二层：网络层（贝叶斯因果图）

用于社交孤立场景，构建完整的因果推断网络：

```
家庭支持不足 ──→ 情绪退缩 ──┐
                           │
同伴连接缺失 ──────────────→├→ 社交退缩 ──→ 孤立风险
                           │
安全威胁暴露 ──→ 情绪退缩 ──┤
                           │
行为退避 ──────────────────→┘
```

**根因节点配置**：

| 节点 | 先验概率 | 影响系数 | 主要证据来源 |
|------|----------|----------|--------------|
| 同伴连接缺失 | 0.20 | 0.92 | 图谱孤立信号、社交观察 |
| 情绪退缩 | 0.16 | 0.78 | 情绪分偏高、AI对话低落 |
| 家庭支持不足 | 0.14 | 0.58 | 家庭困难标签、家校沟通 |
| 安全威胁暴露 | 0.12 | 0.54 | 行为冲突、图谱冲突共现 |
| 行为退避 | 0.10 | 0.48 | 出勤异常、行为事件 |

### 3.3 四项底层规则优化

#### 时间衰减机制

```python
TIME_DECAY_WINDOWS = (
    (7, 1.0),    # 7天内：100%权重
    (30, 0.75),  # 30天内：75%权重
    (90, 0.45),  # 90天内：45%权重
)
# 超过90天：20%权重
```

#### 文本极性分类

```python
TEXT_NEGATIVE_HINTS = ("低落", "焦虑", "孤立", "冲突", ...)
TEXT_POSITIVE_HINTS = ("好转", "缓解", "稳定", "支持", ...)
```

#### 保护性证据机制

正向信号产生负权重，主动降低风险分数：
- 负向文本：保持原权重（增加风险）
- 正向文本：产生负权重（降低风险）

#### 缺失数据机制

数据缺失时生成权重为0的"数据缺口信号"，不影响分数，但提示证据充分度有限。

## 六、人在回路机制

```
AI研判 → 教师确认 → 贝叶斯证据更新 → 概率修正
```

**教师确认状态在贝叶斯计算中的作用**：

| 教师确认状态 | 似然比 | 效果 |
|--------------|--------|------|
| `resolved` (已解决) | 0.35-0.55 | 降低风险概率 |
| `false_alarm` (误报) | 0.15-0.45 | 大幅降低风险概率 |

## 十、公式化表达

```text
# 第一层：线性评分层
S_d = Normalize(Σ w_i · x_i),  i ∈ Signals(d)

# 第二层：贝叶斯辅助层
Odds' = Prior/(1-Prior) × ∏ LR_j
P_d = Odds' / (1 + Odds')

# 第三层：贝叶斯网络层
P(isolation) = OR_gate([
    P(social_withdrawal) × 0.78,
    P(peer_disconnect) × 0.72,
    P(safety_threat) × 0.28
])

# 第四层：混合层
FinalScore_d = λ · S_d + (1 - λ) · P_d
```
