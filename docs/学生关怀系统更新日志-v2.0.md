## 二、核心架构升级

### 2.1 贝叶斯推断系统 v2.0

**文件**: `services/student_care_bayes_service.py`

#### 2.1.1 边际递减机制

解决证据相关性问题，避免多个相关证据过度放大风险：

```python
def apply_diminishing_returns(lrs: list[float], method: str = "sqrt") -> list[float]:
    """
    数学原理：
    - 第1个证据: LR^1
    - 第2个证据: LR^0.707 (衰减)
    - 第3个证据: LR^0.577 (进一步衰减)
    """
```

**效果**: 防止同类证据（如多个出勤异常）过度叠加导致风险虚高。

#### 2.1.2 动态先验

将线性分数融入先验概率，实现历史数据与当前证据的有机结合：

```python
dynamic_prior = base_prior + (1 - base_prior) * linear_score * alpha
# alpha默认0.3
```

#### 2.1.3 保护性证据 (LR < 1)

新增支持降低风险的证据类型：

| 维度 | 保护性证据 | LR值 |
|------|-----------|------|
| emotion | care_talk_positive | 0.50 |
| emotion | assistant_positive_mood | 0.55 |
| social | graph_peer_support | 0.50 |
| social | care_observation_positive_social | 0.55 |
| safety | teacher_review_false_alarm | 0.15 |

#### 2.1.4 最终分数计算

**v1.x**: `final_score = 0.6 * linear_score + 0.4 * posterior`

**v2.0**: `final_score = posterior`（动态先验已融合线性分数）

## 十一、已知问题与限制

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| 情绪维度证据限制 | 仅允许特定来源的证据 | 设计如此，防止误判 |
| 研判超时兜底 | AI响应慢时自动切换规则结果 | 已有兜底机制 |
| 图谱依赖Neo4j | 未配置时图谱信号不可用 | 可关闭NEO4J_ENABLED |
| 中文乱码 | 部分返回内容包含乱码字符 | 已在序列化时处理 |
