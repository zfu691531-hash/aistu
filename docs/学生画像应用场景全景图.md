### 2.2 预警监控场景

#### 2.2.1 风险预警推送

**功能描述**：画像变化触发预警，推送给班主任

**预警规则**：
| 规则 | 触发条件 | 推送对象 |
|------|----------|----------|
| 风险升级 | 风险等级上升 | 班主任 |
| 维度突变 | 单维度分数日增幅>0.2 | 班主任 |
| 安全预警 | safety_score > 0.6 | 班主任 + 年级主任 |
| 批量预警 | 班级高风险学生>5 | 年级主任 |

**API设计**：
```
POST /api/profile-alerts/rules
{
    "name": "高风险学生预警",
    "condition": {
        "field": "risk_level",
        "operator": "in",
        "values": ["high", "critical"]
    },
    "notify": ["head_teacher", "grade_director"],
    "message_template": "学生{student_name}风险等级已升至{risk_level}"
}
```

class ProfileAlertRule(Base, TimestampMixin):
    __tablename__ = "profile_alert_rule"

# 新增：画像预警记录表
class ProfileAlertRecord(Base, TimestampMixin):
    __tablename__ = "profile_alert_record"
    
    id: Mapped[int] = mapped_column(BigInteger, primary_key=True)
    student_id: Mapped[int] = mapped_column(BigInteger, comment="学生ID")

## 六、总结

**学生画像可应用于**：

1. **教育管理** - 分班分组优化、教师工作台、班级对比
2. **预警监控** - 风险预警、批量筛查、趋势追踪
3. **干预支持** - 关怀建议、效果评估、资源分配
4. **家校沟通** - 家长报告、家访排序
5. **数据分析** - 年级统计、学业预测、归因分析

**建议实施顺序**：
```
第一阶段 → 教师工作台 + 预警 + 关怀建议（核心闭环）
第二阶段 → 筛查 + 对比 + 统计 + 家访（扩展能力）
第三阶段 → 效果评估 + 家长报告 + 预测（高级功能）
```
