## 二、系统模块概览

### 2.1 教务核心管理

| 模块 | 功能 | 特色 |
|------|------|------|
| 学生管理 | CRUD、批量导入导出 | 标签、特长、班级关联 |
| 教师管理 | CRUD、班级绑定 | 多班级绑定、职务管理 |
| 班级管理 | CRUD、学生关联 | 实时人数、班主任关联 |
| 成绩管理 | CRUD、批量导入 | 多科目、统计分析 |

### 2.2 AI智能工具矩阵（8个工具）

| 工具 | 交互模式 | 核心功能 |
|------|----------|----------|
| 期末评语生成器 | 单次生成 | 支持批量班级生成，三种风格 |
| 违纪话术教练 | 单次生成 | 三段式话术，学生/家长版本 |
| 公告润色助手 | 单次生成 | 自动纠错，标准格式输出 |
| 校规问答机器人 | 多轮对话 | 基于RAG知识库精准回答 |
| 成绩波动诊断书 | 单次生成 | 自动读取历史成绩，趋势分析 |
| 班会活动策划师 | 单次生成 | 完整教案输出，互动设计 |
| 模拟面试官 | 多轮对话 | AI提问→学生回答→AI点评 |
| 智能分组/分班 | 单次生成 | 蛇形算法均衡分配 |

### 2.3 学生关怀系统

> **系统定位**：不是心理诊断系统，而是基于多源数据的六维风险研判系统

```
六维风险研判体系
├── 情绪状态 (emotion)  18%  → 情绪低落、焦虑、失眠等风险
├── 社交融入 (social)   18%  → 被孤立、同伴互动弱
├── 校园安全 (safety)   18%  → 欺凌、威胁、暴力场景
├── 家庭支持 (family)   18%  → 家庭支持是否不足
├── 学习压力 (study)    16%  → 学业挫败、成绩波动
└── 行为稳定 (behavior) 12%  → 出勤、违纪等节律稳定性
```

## 四、核心功能流程

### 4.1 学生画像计算流程

```
数据采集 → 规则计算 → 贝叶斯修正 → 恶性事件检测 → 存储输出
   │           │           │              │            │
   ├─标签      ├─时间衰减   ├─动态先验    ├─关键词匹配  ├─Profile表
   ├─成绩      ├─文本极性   ├─证据匹配    ├─传导计算    └─Signal表
   ├─出勤      └─保护性证据 ├─边际递减    └─贝叶斯子图
   ├─行为
   ├─观察
   ├─AI摘要
   └─图谱
```

### 4.2 多智能体研判流程

```
上下文构建 → 6专家并行 → 整合智能体 → 后处理增强 → 持久化 → 教师复核
     │           │            │            │           │         │
     ├─画像      ├─emotion   ├─汇聚结果   ├─复核建议   ├─Record  └─证据回传
     ├─信号      ├─social    ├─验证完整   ├─解释亮点   └─JSON
     ├─事实      ├─safety    ├─计算综合   └─恶性标记
     ├─图谱      ├─family
     └─历史      ├─study
                └─behavior
```

## 五、数据库设计亮点

### 5.1 无外键设计

- **决策**：业务层保证数据一致性
- **优势**：性能高、灵活性强、易于扩展

## 六、安全设计

| 安全项 | 方案 | 说明 |
|--------|------|------|
| 密码存储 | bcrypt哈希 | 禁止MD5，自动加盐 |
| 身份认证 | JWT Token | 请求头Authorization携带 |
| 接口鉴权 | FastAPI依赖注入 | 未授权返回401 |
| 数据权限 | 业务层按角色+关联ID过滤 | 三级权限隔离 |
| SQL注入 | SQLAlchemy ORM参数化查询 | 自动防注入 |
| 用户输入 | Pydantic Schema双重校验 | 前端+后端校验 |

## 七、项目特色

| 特色 | 说明 |
|------|------|
| **双层贝叶斯架构** | 辅助层(Odds-LR) + 网络层(因果图推断) |
| **边际递减机制** | 处理证据相关性，LR^(1/√n) |
| **时间衰减窗口** | 近期事件权重更高 |
| **文本极性分析** | 自动识别正向/负向信号 |
| **恶性事件传导** | 4条因果边定义风险传导路径 |
| **人在回路** | 教师确认作为贝叶斯证据 |
| **LangGraph编排** | 6专家并行 + 整合智能体 |
| **无外键设计** | 业务层保证数据一致性 |

```bash
# 前端启动
cd frontend
npm install
npm run dev           # 访问 http://localhost:5173
```
