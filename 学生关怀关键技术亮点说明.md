## 亮点一：双层贝叶斯架构

### 1.1 为什么需要双层设计？

| 层级 | 目的 | 实现位置 | 特点 |
|------|------|----------|------|
| **通用贝叶斯辅助层** | 对各维度做轻量后验修正 | `student_care_bayes_service.py` | 单维度、多证据、动态先验 |
| **贝叶斯网络专项层** | 处理因果链推断与传播分析 | `student_care_isolation_service.py` | 多节点、有向边、条件概率表 |

### 1.2 核心数学公式

**后验概率计算**（通用层）：

$$
P(R|E_1, E_2, ..., E_n) = \frac{P(R) \times \prod_{i=1}^{n} LR_{adj}(E_i)}{P(R) \times \prod_{i=1}^{n} LR_{adj}(E_i) + (1 - P(R))}
$$

其中：
- $P(R)$ = 维度先验概率（从配置读取）
- $LR(E_i)$ = 证据 $E_i$ 的似然比
- $LR_{adj}(E_i)$ = 边际递减调整后的似然比

**边际递减公式**（处理证据相关性）：

$$
LR_{adj}(E_i) = LR(E_i)^{\frac{1}{\sqrt{i}}}
$$

**实现代码** (`student_care_bayes_service.py:57-95`)：

```python
def apply_diminishing_returns(lrs: list[float], method: str = "sqrt") -> list[float]:
    """对似然比应用边际递减，处理证据相关性问题。
    
    数学原理：
    贝叶斯定理假设证据条件独立，但实际场景中多个证据往往相关性较高。
    边际递减通过降低后续证据的权重来缓解这个问题。
    
    Args:
        lrs: 似然比列表
        method: 递减方法
            - "sqrt": 第n个证据权重 = LR^(1/√n)，推荐
            - "log": 边际效应线性递减
            - "none": 不应用递减
    """
    adjusted = []
    for i, lr in enumerate(lrs, start=1):
        if method == "sqrt":
            # 第1个证据: LR^1, 第2个: LR^0.707, 第3个: LR^0.577...
            adj_lr = lr ** (1.0 / math.sqrt(i))
        elif method == "log":
            adj_lr = 1.0 + (lr - 1.0) / i
        else:
            adj_lr = lr
        adjusted.append(max(adj_lr, 0.05))  # 最小LR=0.05
    return adjusted
```

## 亮点二：边际递减机制

### 2.1 问题背景

贝叶斯定理的核心假设是**证据条件独立**：

$$
P(E_1, E_2 | R) = P(E_1 | R) \times P(E_2 | R)
$$

但实际场景中，多个证据往往存在**相关性**：
- 同一学生连续3次迟到 → 第3次的边际信息量 < 第1次
- 同一冲突事件被多个表记录 → 不应线性叠加

### 2.2 解决方案对比

| 方法 | 公式 | 特点 | 适用场景 |
|------|------|------|----------|
| **平方根衰减** (推荐) | $LR_{adj} = LR^{1/\sqrt{i}}$ | 第n个证据权重指数衰减 | 证据相关性高的场景 |
| **对数衰减** | $LR_{adj} = 1 + (LR-1)/i$ | 边际效应线性递减 | 证据相关性中等 |
| **无衰减** | $LR_{adj} = LR$ | 不处理相关性 | 证据完全独立 |

### 2.3 效果演示

假设学生有3条同类证据，原始 LR = [2.5, 2.5, 2.5]：

```python
>>> apply_diminishing_returns([2.5, 2.5, 2.5], method="sqrt")
[2.5, 2.10, 1.82]  # 第2条衰减16%，第3条衰减27%

# 无边际递减时的后验（假设prior=0.15）
>>> posterior = (0.15 * 2.5 * 2.5 * 2.5) / (0.15 * 2.5 * 2.5 * 2.5 + 0.85)
0.735  # 过度放大

# 有边际递减时的后验
>>> posterior = (0.15 * 2.5 * 2.10 * 1.82) / (0.15 * 2.5 * 2.10 * 1.82 + 0.85)
0.628  # 更合理的估计
```

## 亮点三：时间衰减机制

### 3.1 设计原理

校园场景中，**近期事件比历史事件更具判断价值**。系统采用分段线性衰减：

```python
TIME_DECAY_WINDOWS = (
    (7, 1.0),    # 近7天：权重100%
    (30, 0.75),  # 8-30天：权重75%
    (90, 0.45),  # 31-90天：权重45%
)
```

### 3.2 衰减计算实现

```python
def calculate_time_decay(event_date: date, reference_date: date = None) -> float:
    """计算事件的时间衰减权重"""
    if reference_date is None:
        reference_date = date.today()
    
    days_ago = (reference_date - event_date).days
    
    for window_days, weight in TIME_DECAY_WINDOWS:
        if days_ago <= window_days:
            return weight
    
    return 0.20  # 超过90天：最小权重20%
```

### 3.3 与边际递减的协同

时间衰减与边际递减**独立作用**，共同影响最终权重：

$$
w_{final} = w_{base} \times \text{time\_decay}(t) \times \text{diminishing\_factor}(i)
$$

```python
class AgentState(TypedDict):
    student_id: int
    student_info: dict
    signals: list[dict]
```

### 4.3 专家节点实现

```python
def create_expert_node(dimension: str):
    """创建维度专家节点"""
    async def expert_node(state: AgentState) -> dict:
        # 1. 过滤该维度的证据
        dim_signals = [s for s in state["signals"] 
                       if s.get("dimension") == dimension]
        
        # 2. 获取贝叶斯修正结果
        bayes_score = state["bayes_results"].get(dimension, {})
        
        # 3. 构建专家提示词
        prompt = ChatPromptTemplate.from_messages([
            ("system", EXPERT_PROMPTS[dimension]),
            ("user", """
            学生信息：{student_info}
            该维度证据：{signals}
            贝叶斯评分：{bayes_score}
            
            请分析该学生在{dimension_label}维度的风险情况。
            """)
        ])
        
        # 4. 调用LLM
        chain = prompt | ai_client
        result = await chain.ainvoke({
            "student_info": state["student_info"],
            "signals": dim_signals,
            "bayes_score": bayes_score,
            "dimension_label": DIMENSION_LABELS[dimension],
        })
        
        # 5. 返回状态更新
        return {
            "dimension_analyses": {
                dimension: {
                    "analysis": result.content,
                    "risk_level": bayes_score.get("risk_level"),
                    "evidence_count": len(dim_signals),
                }
            }
        }
    
    return expert_node
```

### 4.4 为什么不用单一大模型？

| 对比维度 | 单一大模型 | 多智能体架构 |
|----------|------------|--------------|
| **信息处理** | 一次性处理所有信息，易丢细节 | 各专家聚焦单一维度 |
| **输出形式** | 只有总评，不利于分维理解 | 保留分维度结论 |
| **证据压制** | 强维度压制弱维度 | 各维度独立分析 |
| **可解释性** | 黑箱输出 | 分维度证据链清晰 |
| **故障隔离** | 单点失败 | 专家失败不影响其他维度 |

## 亮点五：保护性证据机制

### 5.1 设计理念

传统风险系统**只累加负向证据**，导致风险分数"只升不降"。本系统显式处理保护性证据：

| 证据类型 | LR范围 | 作用 | 示例 |
|----------|--------|------|------|
| **负向证据** | LR > 1 | 提高后验概率 | 冲突事件、成绩下滑 |
| **中性证据** | LR = 1 | 无影响 | 无关记录 |
| **保护性证据** | LR < 1 | 降低后验概率 | 情绪好转、同伴支持 |

## 亮点六：安全披露捕获

### 6.1 问题背景

校园场景中，很多风险线索存在于**非结构化文本**中：
- AI对话："我害怕回班里"
- 家校沟通："孩子身上有淤青"
- 出勤备注："不想上学"

传统规则系统**只能处理结构化事件表**，会漏掉大量早期信号。

```python
class ReviewStatus(str, Enum):
    PENDING = "pending"          # 待复核
    CONFIRMED = "confirmed"      # 确认真风险
    FALSE_ALARM = "false_alarm"  # 确认误报
    RESOLVED = "resolved"        # 已解决
    IN_PROGRESS = "in_progress"  # 处理中
```

### 8.3 评估指标

| 指标 | 计算方式 | 用途 |
|------|----------|------|
| **规则命中率** | 命中规则数 / 总规则数 | 评估规则覆盖度 |
| **复核一致率** | 教师确认数 / 总研判数 | 评估系统准确性 |
| **误报率** | 误报数 / 总研判数 | 评估规则敏感度 |
| **场景分布** | 各维度研判数占比 | 识别高风险场景 |
