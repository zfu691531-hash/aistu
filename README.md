# 校园 AI 教务助手平台

校园 AI 教务助手平台面向中小学教务与学生发展场景，整合基础教务管理、学生关怀研判、学生画像应用、智能分组分班、校规知识服务与多种 AI 教务工具，支持学生、教师、校务管理员三级角色协同使用。

本项目当前已经形成两条主线能力：

1. 教务管理主线：学生、教师、班级、成绩、校规等基础数据的统一管理。
2. 智能辅助主线：围绕学生画像、风险研判、AI 生成和知识检索形成闭环应用。

## 项目定位

项目不是单一的“教务系统”，也不是单一的“AI 助手”，而是把结构化校园数据与智能分析能力放在同一个工作台里。老师可以在同一套系统中完成：

- 学生信息维护与班级管理
- 成绩与过程性数据查看
- 学生关怀画像与风险研判
- 标签审核与规则沉淀
- 校规检索与问答
- AI 教务工具调用

## 核心模块

| 模块 | 主要能力 | 面向角色 |
| --- | --- | --- |
| 学生管理 | 学生档案、班级归属、过程记录汇总 | 教师、校务管理员 |
| 教师管理 | 教师信息、任课关系、班主任绑定 | 校务管理员 |
| 班级管理 | 班级信息、班主任关联、学生归班 | 教师、校务管理员 |
| 成绩管理 | 考试批次、学科成绩、趋势对比 | 教师、校务管理员 |
| 学生关怀 | 风险信号抽取、学生画像、智能研判、教师复核 | 教师 |
| 标签管理 | 标签定义、标签审核、标签沉淀 | 教师、校务管理员 |
| 智能分组分班 | 教师分组、校务均衡分班 | 教师、校务管理员 |
| 校规服务 | 校规录入、检索问答、规则引用 | 教师、学生、校务管理员 |
| AI 教务工具 | 评语生成、成绩诊断、公告润色、班会策划等 | 教师 |

## 技术栈

### 后端

- FastAPI：提供 REST API 与接口文档
- SQLAlchemy：负责 ORM 与数据库访问
- MySQL：存储业务主数据
- LangChain 与 LangGraph：编排智能体和多步推理流程
- Milvus：承载校规向量检索
- Neo4j：支持关系增强与图谱分析

### 前端

- Vue 3：构建页面与组件
- Vite：提供开发与构建能力
- Element Plus：后台管理界面组件
- Pinia：管理用户状态与全局数据
- Axios：统一接口调用和拦截处理

## 当前能力概览

### 基础教务

- 学生、教师、班级、成绩等基础模块完整可用
- 支持常规增删改查、关联查询与部分导入导出
- 前后端角色权限区分明确

### 学生关怀

- 基于成绩、考勤、行为、家校沟通、AI 摘要等多源数据生成学生画像
- 支持六维风险分析：情绪、社交、安全、家庭、学习、行为
- 支持教师复核、误报标记、真风险确认与跟踪闭环
- 支持研判评估统计，查看规则命中与复核效果

### AI 辅助

- 提供多种教学和班级管理场景下的 AI 工具
- 支持校规知识检索与问答
- 支持学生分组分班等半自动智能辅助流程

## 运行方式

### 后端

1. 安装依赖。
2. 配置 `backend/.env`。
3. 启动 FastAPI 服务。

推荐在 `backend/` 目录执行常规开发命令，例如安装依赖、运行服务、执行脚本和测试。

### 前端

1. 安装依赖。
2. 配置接口地址。
3. 启动开发服务或执行构建。

推荐在 `frontend/` 目录执行常规开发命令，例如本地开发和生产构建。
