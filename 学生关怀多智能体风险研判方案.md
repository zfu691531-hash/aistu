# 学生关怀多智能体风险研判系统

> 面向校园场景的学生关怀智能分析模块。通过多智能体协同分析学生在情绪、社交、安全、家庭、学习、行为六个维度上的风险线索，辅助班主任更早发现需要关注的学生。

---

## 🌟 技术亮点

本项目采用 **LangChain + LangGraph + 贝叶斯推断 + 人在回路 + RAG增强 + 图谱增强** 实现多智能体协同研判，具有以下技术亮点：

**1. 多智能体并行架构**
- 使用 LangGraph StateGraph 实现 6 个专家智能体并行执行
- 总耗时≈单个专家耗时（而非串行叠加），大幅提升处理效率
- 实现：`backend/services/student_care_agent_service.py`（48.79KB）

**2. 结构化工作流编排**
- 可视化工作流：START → 6个专家节点（并行）→ 整合节点 → END
- 节点级容错：每个专家独立捕获异常，失败不影响其他专家
- 兜底机制：模型超时或JSON解析失败时自动使用规则引擎结果

**3. 状态汇聚与传递**
- 使用 TypedDict 定义全局状态，确保类型安全
- `Annotated[list[dict], operator.add]` 自动汇聚并行节点结果

**4. 贝叶斯辅助推断层**
- 对安全、情绪、家庭、社交四个高敏感维度启用贝叶斯概率修正
- 通过似然比累乘实现"关键证据出现后风险概率显著上升"
- 混合公式：`final = α × linear_score + (1-α) × posterior`
- 实现：`backend/services/student_care_bayes_service.py`（11.13KB）

**5. 人在回路机制（Human-in-the-Loop）**
- 教师作为关键环节参与决策流程，对AI结果进行确认、修正
- 教师确认状态作为贝叶斯证据参与下轮概率更新
- 确保教育场景下的人类主体性和责任归属清晰
- 接口：`POST /api/student-care/agent-eval-review/{record_id}`

**6. 安全披露自动捕获**
- 学生AI助手具备关键词检测功能，自动捕获"被打"、"受伤"、"霸凌"等安全信号
- 自动生成关怀信号（safety:0.75, emotion:0.4）并触发画像重算
- 关键词：`被打, 打我, 受伤, 流血, 欺负, 霸凌, 威胁, 校园暴力`
- 实现：`backend/services/assistant_service.py`（43.65KB）



**7. 图谱关系增强层**
- Neo4j图数据库存储学生关系网络
- 识别冲突共现、社交孤立等关系型线索
- 图信号回流到画像、贝叶斯证据和Agent prompt
- 实现：`backend/services/student_care_graph_service.py`（20.27KB）



**技术栈：**
- 后端：FastAPI + SQLAlchemy + Pydantic V2
- AI：`langchain-core`, `langgraph`, OpenAI兼容API
- 向量数据库：Milvus
- 图数据库：Neo4j（可选）
- 前端：Vue 3 + Element Plus + Vite

**核心实现文件：**
- `backend/services/student_care_agent_service.py` - 多智能体引擎
- `backend/services/student_care_bayes_service.py` - 贝叶斯辅助层
- `backend/services/student_care_service.py` - 学生关怀画像服务
- `backend/services/student_care_graph_service.py` - 图谱关系增强

## 一、功能定位

**本系统是：**
- 学生风险线索识别系统
- 班主任关怀辅助决策系统
- 基于多源数据的六维研判系统

**本系统不是：**
- 心理诊断系统

**核心输出：**
- 六维风险雷达图（情绪/社交/安全/家庭/学习/行为）
- 每个维度的风险分数、依据摘要、得分明细
- 综合风险等级和建议跟进动作

## 二、技术架构

### 2.1 多智能体协同架构

采用 **LangGraph StateGraph** 实现 `6 个维度专家智能体（并行） + 1 个整合智能体` 的工作流：

```
START
  ├─ emotion_agent (情绪专家) ──┐
  ├─ social_agent (社交专家)  ──┤
  ├─ safety_agent (安全专家)  ──┤
  ├─ family_agent (家庭专家)  ──┼──> integration_agent ──> END
  ├─ study_agent (学习专家)   ──┤
  └─ behavior_agent (行为专家)──┘
```

**核心代码结构：**

```python
# 状态定义
class StudentCareGraphState(TypedDict, total=False):
    prompt_payload: dict        # 提示词负载
    profile: dict               # 学生画像数据
    signals: list[dict]         # 风险信号列表
    expert_outputs: Annotated[list[dict], operator.add]  # 并行结果汇聚
    result: StudentCareAgentResult | None  # 最终研判结果

# 工作流构建
@lru_cache(maxsize=1)
def _build_student_care_agent_graph():
    workflow = StateGraph(StudentCareGraphState)
    
    # 添加6个维度专家节点（并行执行）
    for dimension in ["emotion", "social", "safety", "family", "study", "behavior"]:
        node_name = f"{dimension}_agent"
        workflow.add_node(node_name, _build_dimension_expert_node(dimension))
        workflow.add_edge(START, node_name)  # 并行启动
    
    # 添加整合节点
    workflow.add_node("integration_agent", _integration_agent_node)
    workflow.add_edge(expert_node_names, "integration_agent")
    workflow.add_edge("integration_agent", END)
    
    return workflow.compile(name="student_care_multi_agent")
```

**核心优势：**
- ✅ **并行执行**：6个专家同时调用大模型，总耗时≈单个专家耗时
- ✅ **状态汇聚**：`Annotated[list[dict], operator.add]` 自动汇聚并行结果
- ✅ **节点级容错**：每个专家独立捕获异常，失败不影响其他专家
- ✅ **兜底机制**：模型超时或JSON解析失败时自动使用规则引擎结果

### 2.2 六维风险研判体系

| 维度 | 权重 | 研判内容 | 启用贝叶斯 |
|------|------|----------|------------|
| 情绪状态 emotion | 18% | 情绪低落、焦虑、疲惫、失眠等风险 | ✅ |

| 校园安全 safety | 18% | 欺凌、威胁、暴力、恐惧场景 | ✅ |
| 家庭支持 family | 18% | 家庭支持是否不足 | ✅ |
| 学习压力 study | 16% | 学业挫败、考试焦虑、成绩波动 | ❌ |
| 行为稳定 behavior | 12% | 行为节律是否稳定（出勤、违纪等） | ❌ |

### 2.3 图谱关系增强层

```
┌─────────────────────────────────────────────────────────────┐
│                    图谱增强架构                               │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │ 关系型主库    │ -> │ Neo4j图数据库 │ -> │ 图信号生成   │  │
```

**图信号类型：**
- `graph_conflict_cooccurrence`：冲突共现（与有冲突记录的同学存在关联），似然比1.9

**图谱节点：** Student（学生）、Class（班级）、Event（行为事件）

**图谱关系：** SAME_CLASS（同班）、INVOLVED_IN（关联事件）、CONFLICT_COOC（冲突共现）

### 5.4 兜底机制

当模型超时、返回无法解析的JSON、或心理维度高分但证据不足时，自动使用规则引擎生成的画像分数，并标记 `fallback=True`

```python
def _build_fallback_result(profile: dict, signals: list[dict]) -> StudentCareAgentResult:
    """规则引擎兜底结果生成"""
    dimensions = []
    for dimension in DIMENSION_LABELS.keys():
        score = float(profile.get(f"{dimension}_score") or 0)
        evidence = [item["signal_text"] for item in signals if item["dimension"] == dimension][:2]
        summary = "暂无明显风险线索" if not evidence else "结合近期信号需要持续关注"
        dimensions.append(StudentCareAgentDimension(
            dimension=dimension,
            score=score,
            risk_level=_risk_level(score),
            summary=summary,
            evidence=evidence,
        ))
    return StudentCareAgentResult(
        overall_score=profile.get("overall_risk", 0),
        overall_level=profile.get("risk_level", "low"),
        suggestions=["建议持续观察并保持常规关怀。"],
        dimensions=dimensions,
    )
```

**算法：** 蛇形分配算法（成绩排序后交替分配到各组）

```python
def _balance_group(students: List[dict], group_count: int) -> List[dict]:
    sorted_students = sorted(students, key=lambda x: x["avg_score"], reverse=True)
    groups = [{"students": [], "avg_score": 0} for _ in range(group_count)]
    
    direction = 1
    group_index = 0
    for student in sorted_students:
        groups[group_index]["students"].append(student)
        group_index += direction
        # 蛇形转向
        if group_index >= group_count:
            group_index = group_count - 2
            direction = -1
        elif group_index < 0:
            group_index = 1
            direction = 1
    return groups
```

### 9.2 核心公式

```python
def _calculate_posterior(prior: float, likelihood_ratios: list[float]) -> float:
    # 先验概率转赔率
    odds = prior / (1 - prior)
    # 累乘似然比
    for lr in likelihood_ratios:
        odds *= max(lr, 0.01)
    # 赔率转回概率
    return odds / (1 + odds)

# 混合公式
final_score = blend_alpha * linear_score + (1 - blend_alpha) * posterior
```

### 13.1 项目简介版

本项目面向高中校园学生关怀场景，构建了一套"规则评分 + 贝叶斯辅助更新 + 多智能体研判 + 人在回路确认 + RAG增强 + 图谱增强"的学生风险研判系统。系统从校内数据中提取风险信号，形成六维关怀画像。在安全、情绪、家庭、社交四个高敏感维度，通过贝叶斯推断对关键证据进行概率修正。系统创新性地引入RAG混合检索增强校规问答能力、Neo4j图谱识别关系型风险线索，以及智能分组助手减轻教师事务负担。人在回路机制确保教师对AI研判结果拥有最终裁量权，教师确认结果会作为贝叶斯证据参与下轮概率更新。

### 13.2 技术路线版

本项目采用"事实采集层 - 信号抽取层 - 线性评分层 - 贝叶斯辅助更新层 - 多智能体解释层 - 人在回路确认层"的分层技术路线。创新点包括：
1. LangGraph编排6专家并行研判
2. 贝叶斯似然比累乘概率修正
3. 教师确认闭环学习
4. RAG混合检索（BM25 + 稀疏向量 + 密集向量）
5. Neo4j图谱关系增强（冲突共现、社交孤立）
6. 智能分组蛇形算法

### 13.3 公式化表达

```text
# 线性评分层
S_d = Normalize(Σ w_i · x_i),  i ∈ Signals(d)

# 贝叶斯辅助层
Odds' = Prior/(1-Prior) × ∏ LR_j
P_d = Odds' / (1 + Odds')

# 混合层
FinalScore_d = λ · S_d + (1 - λ) · P_d

# RAG检索融合
FusedScore = 0.55·BM25 + 0.20·TitleBoost + 0.20·Sparse + 0.05·Dense
```
